# Weave — 아이디어톤 기획안 (재작성 v2)

> **서비스명**: **Weave (위브)**
> **대회**: 멋사대학 아이디어톤 (D-7, 마감 2026-05-18 15:00)
> **주제**: "AI로 [ ]을/를 없앤다면?"
> **확정 슬로건**: **"AI로 시작에 헤매는 시간을 없앤다면?"**

### 서비스 한 줄 소개
시작에 헤매는 0→1 구간을 AI가 **한 줄로 안내**합니다.
흩어져 있던 아이디어·피드백·합의를 **하나의 흐름으로 엮어**, 익숙한 도구(Notion·Slack·Figma)로 **그대로 떠나보냅니다.**

> *Weave — 아이디어가 자유롭게 흐르고(Wave) 유기적으로 엮이는(Weave) 과정.*

---

## 0. 한 줄 컨셉

> **두서없는 아이디어 → 팀 합의된 1페이지 기획안.**
> **0→1 구간만** AI가 책임지고, **그 다음은 익숙한 도구(Notion·Slack·Figma 등)에 넘긴다.**

새 도구를 또 익히게 하지 않는다. **시작만** 도와주고 빠진다.

---

## 1. 왜 이 문제인가 (Problem)

우리도 이 아이디어톤을 준비하면서 직접 겪었다.

- **메타 합의**(언제까지 제출할까)는 → 카카오톡에서
- **아이디어 정리**는 → 노션·피그마에서
- **피드백**은 → 노션·피그마에서 (그런데 **알림이 안 와서 놓침**)
- **투표**는 → 다시 카카오톡 투표 기능에서

문서는 파편화되고, 피드백은 정리가 안 돼 확인이 어렵고, 투표는 또 다른 앱에서. **여러 앱을 왔다갔다하며 흐름이 끊긴다.** 처음 해보는 사람은 어디서 뭘 해야 할지 몰라 **우왕좌왕**한다.

각 앱은 0→1의 **한 조각씩만** 책임진다. **전체 흐름을 책임지는 도구는 없다.**
- ChatGPT는 개인 정리는 도와주지만 **팀 합의**를 못 하고,
- Notion·Jira는 **이미 방향이 잡힌 팀**을 위한 것이다.

**0→1의 전체 흐름을 한 줄로 안내하는 도구**가 없다.

### 같은 통증, 세 가지 모습 — 타깃 시나리오

| 시나리오 | 사용자 | 공통 페인 | 추가 페인 |
|---|---|---|---|
| **A. 아이디어톤·해커톤** | 처음 만난 팀, 마감 48h~2주 | 도구 파편화·우왕좌왕 | **심사 통과** 가능성 검증, 발표 산출물 |
| **B. 사이드 프로젝트** | 친구·동료끼리 가볍게 시작 | 도구 파편화·결론 못 냄 | 본업 있어 **비동기 진행**, 흐지부지 방지 |
| **C. 창업·MVP 검토** | 공동창업자·예비 창업팀 | 도구 파편화·합의 늘어짐 | **시장성·BM 검증**, 투자자 관점 |

→ 세 시나리오 모두 **0→1의 동일한 통증**. Weave는 사용자가 선택한 **프로젝트 유형에 따라 AI 페르소나·평가 기준·산출물 형식이 자동 적응**한다.

## 2. 컨셉 — "다리(Bridge)" 포지셔닝

```
[혼란스런 머릿속]                                                    [실행]
       │                                                              │
       │     ◇━━━━━━━━━━━━ Weave ━━━━━━━━━━━━◇                       │
       │     0 ─── 정리 ─── 합의 ─── 1페이지 기획안                      │
       │                                       │                      │
       │                                       └─→ Notion / Slack /   │
                                                   Figma / Jira /     │
                                                   Google Docs ...    │
```

- **우리가 하는 것**: 머릿속의 0 → 팀 합의된 1페이지 기획안
- **우리가 안 하는 것**: 일정·작업관리·디자인·코드·문서 협업 → 기존 도구로 export
- **승부수**: 0→1을 가장 빠르게 끝내고, 결과물을 **그대로 Notion/Slack에 붙여넣기 가능한 형태**로 떨궈준다

## 3. 슬로건 — 확정

> ### **"AI로 시작에 헤매는 시간을 없앤다면?"**

### 슬로건이 담은 것

| 단어 | 가리키는 것 |
|---|---|
| **시작에** | 0→1 구간을 명확히 지칭 — 1→N으로 가는 팀이 아닌, **시작하는 팀**이 타깃 |
| **헤매는** | "어디 적었더라" / "알림 못 봐서 놓침" / "투표는 어디서?" / "처음이라 뭐부터?" — 모든 페인을 한 단어로 |
| **시간을 없앤다** | AI가 사용자의 일을 **대신**하는 게 아니라, 사용자의 흐름에서 **헤매는 구간만 잘라낸다** |

### 차별성 (심사 기준 20%)

다른 팀의 메시지: *"AI가 X를 대신해줘요"* (글쓰기·그림·코드…)
우리의 메시지: *"AI가 사용자를 **데려가줘요**"* — 사람의 일은 그대로 두고, 흐름만 매끄럽게.

→ **타깃의 자기 발견 + 메타 차별화**. 100초 영상 첫 라인부터 같은 호흡으로 풀어진다.

## 4. AI 활용의 깊이 (차별성 20% 핵심)

단순 프롬프트 1회 호출은 **"ChatGPT로도 되잖아"** 반박을 못 막는다. 다음 세 축으로 차별화:

### 4-1. 멀티 페르소나 시뮬레이션 (가상 팀원) — 컨텍스트 적응형

**구조**: **공통 3명 + 컨텍스트별 1명 = 매 세션 4명**.
실제 팀원이 응답하기 전에 가상 페르소나가 **먼저** 비평하고, 솔로 모드에서도 흐름이 안 멈추게 한다.

#### 공통 3명 (모든 컨텍스트)

| 역할 | 성격 | 평가 우선순위 | 말투 | 샘플 한 줄 |
|---|---|---|---|---|
| **① 사용자 입장** | 본인 시간·돈 관점, 솔직 | "내가 진짜 쓸까?" / "왜 필요해?" / "쓰기 귀찮지 않아?" | 짧고 직설, 의문문 많음 | *"이거 매주 한 번씩 켤 만큼 좋아? 카톡으로도 되는데."* |
| **② 회의적 동료** | 모든 가정에 "왜?"를 4번 묻는 분석가 | "이게 진짜 문제인가?" / "이미 X가 해결하지 않나?" / "근거?" | 차분·논리적, 약간 까칠 | *"문제 정의는 좋은데, 그게 우선순위가 높은 문제인지 데이터가 없네요."* |
| **③ 현실 체크** | 실현 가능성·일정·자원 균형 감각 | "N일 안에 만들 수 있어?" / "기술·인력·예산 가능?" | 차분·실무적, 숫자 자주 | *"D-7 안에 4개 기능은 무리. 핵심 1개만 우선 배포하면 가산점은 확보."* |

#### 컨텍스트별 +1명

| 컨텍스트 | 역할 | 성격 | 평가 우선순위 | 샘플 한 줄 |
|---|---|---|---|---|
| 아이디어톤·해커톤 | **④A 심사위원** | 대회 통과 관점, 차별성·전달력 | "차별성?" / "5분 안에 와닿나?" / "심사 기준 부합?" | *"문제 정의 8/10, 차별성 6/10. '왜 우리가 처음 했나'를 명확히."* |
| 사이드 프로젝트 | **④B 시간 관리자** | 흐지부지 방지, 작은 단위 쪼개기 | "주말 N시간으로 가능?" / "최소 단위 첫 출시?" / "지속 가능?" | *"이번 주 토요일 2시간으로 끝낼 수 있는 건 'X'까지. 그 다음은 다음 주."* |
| 창업·MVP 검토 | **④C VC (투자자)** | 시장·BM·경쟁·성장 | "TAM은?" / "BM·Moat?" / "경쟁 우위?" / "스케일?" | *"TAM 추정이 약합니다. 처음 1000명 유료 전환 경로를 보여주세요."* |

#### 차별화 포인트
- 단일 LLM은 **자기 자신이 동의하는** 비평을 못 함 — 멀티 에이전트는 **서로 충돌하는** 관점을 동시에 제시
- "ChatGPT로도 되잖아?" 반박을 정면 격파
- 솔로 사용자도 4명짜리 팀처럼 진행 가능

> **출시 범위**: MVP(D-7)는 **아이디어톤 컨텍스트 1개만** 동작 (공통 3 + 심사위원). 사이드·창업 컨텍스트는 **v1.1(본선 진출 후)** 추가.

### 4-2. 의견 충돌 자동 중재
실제 팀원 피드백이 들어오면:
- "A는 실현 가능성 높다고, B는 낮다고 했음" → **양쪽 근거를 추출하고 검증 질문을 자동 생성**
- 합의된 부분/충돌하는 부분을 **시각적으로 분리** → 합의된 건 그대로 통과, 충돌만 추가 논의
- 이게 우리의 **킬러 화면**: "AI가 우리 팀의 갈등을 좁혀줬다"

> 두 기능 다 **단일 LLM 호출로는 불가능한 흐름**. 데모에서 1분 안에 보여줄 수 있다.

### 4-3. AI 아이디어 시각화 (멀티모달)

아이디어 제출 시 **자동으로 컨셉 이미지** 생성 (DALL-E 3 / Imagen 3).

- **목적**: 텍스트만으로 안 보이는 추상적 아이디어를 시각화 → 팀원이 0.5초 안에 "감 잡기" → 합의 가속
- **스타일 고정**: *"Minimalist line illustration, indigo + mustard accents on cream paper, flat style, no text"* — Thread 팔레트 자동 통일
- **위치**: 모아보기 카드 썸네일 / 상세 페이지 hero 이미지 / 카톡 공유 카드 / 최종 기획안 PDF
- **컨트롤**: 자동 생성 후 "다시 생성"(1회) 또는 "끄기" 토글
- **차별화 포인트**: 텍스트 LLM 단일 호출 ≠ 멀티모달 (텍스트 + 이미지) 워크플로우
- **부가 효과**: **카톡 공유 시 썸네일 = 좋아요 캠페인 클릭률 직결** (시장 반응도 20%)

> ⛔ 코드/풀 프로토타입 디자인 생성은 안 함 — "익숙한 도구로 떠나보낸다" 컨셉 유지.

## 5. 핵심 사용자 흐름 (5분 데모용)

```
[1] 팀장 온보딩 (1분)
    └─ 카카오 로그인 → 주제·마감일·심사 기준 설정 → 카톡으로 팀원 초대

[2] 각자 두서없이 입력 (비동기, 5~10분)
    └─ "이런 거 어때?" 자유 텍스트 / 음성 메모도 가능 (선택)

[3] AI 1차 정리 + 가상 페르소나 비평 (즉시)
    └─ 4개 페르소나가 각자 다른 각도에서 질문·반론

[4] 팀원 피드백 → AI 충돌 중재 (10~30분)
    └─ 합의/충돌 분리, 충돌만 모아 빠르게 결정

[5] 1페이지 기획안 출력 + 외부 도구로 내보내기
    └─ Markdown / Notion 페이지 / Slack 메시지 / 카톡 공유
```

총 소요시간 목표: **30분~1시간 안에 0→1 완료**.

## 6. 핵심 기능 (MVP 4개로 축소)

| # | 기능 | 설명 | AI 깊이 |
|---|---|---|---|
| F1 | **자유 입력** | 팀장이 주제·기간 설정 후 팀원이 자유 텍스트로 아이디어/생각 작성 | LLM이 표현 정리·중복 병합 |
| F2 | **AI 정리 + 가상 페르소나 비평** | 입력을 5개 항목(문제/타깃/핵심기능/차별점/리스크)으로 정리 + 4명의 페르소나 AI가 비평 | **멀티 에이전트** ★ |
| F3 | **팀원 피드백 + 충돌 중재** | 항목별 동의/반론. AI가 합의/충돌 자동 분리, 충돌만 빠르게 결정 | **충돌 탐지·중재** ★ |
| F4 | **1페이지 기획안 + Export** | Markdown 기획안 자동 생성. MVP는 **클립보드 복사 + 카톡 카드 공유**, Plus부터 **Notion / Slack 직접 push** (섹션 9 참조) | LLM 요약 |
| F5 | **AI 아이디어 시각화 (멀티모달)** ★ | 아이디어 제출 시 컨셉 이미지 자동 생성. 카드 썸네일·hero 이미지·카톡 공유 카드에 활용 (섹션 4-3 참조) | **이미지 생성 (DALL-E 3 / Imagen 3)** ★ |

> v1 미포함: 일정 보드, 작업 분배, PPT 디자인, 발표 스크립트, 다중 아이디어 비교(필요 시 v2).

## 7. 온보딩 — 팀장이 처음 한 번만

팀원 초대 **전**에 팀장이 채우는 7개 항목:

1. **프로젝트 유형** ★ — 아이디어톤·해커톤 / 사이드 프로젝트 / 창업·MVP / 학교 과제 / 기타
   → 이 선택이 **AI 페르소나 세트·평가 기준·산출물 템플릿**을 자동 결정
2. **프로젝트 주제·키워드** (예: 멋사 아이디어톤, B2C 음식 추천 SaaS)
3. **마감일/목표일** (자동 카운트다운, 선택)
4. **평가/심사 기준** (대회는 입력, 사이드는 자유, 창업은 BM·시장성 기본)
5. **팀 규모·역할** (개발자 N, 디자이너 N…)
6. **피드백 공개 정책** ★ — 팀장이 팀 단위로 결정 (개인 토글 X)
   - (a) **하이브리드** *(기본값 권장)* — 정량 평가(별점·동의/반론)는 익명, 글 코멘트는 실명
   - (b) **전체 익명** — 친한 사이·후배가 위축되는 팀에 적합
   - (c) **전체 실명** — 책임감·추적성이 중요한 팀에 적합
   - 모든 팀원이 같은 룰을 따르므로 "나만 익명"의 어색함 없음. 변경 시 전원 알림.
7. **산출물 형태** (PPT, 영상, 1페이지 기획서, 린 캔버스 등)
8. **톤** (캐주얼/포멀, 한국어/영어)

→ 이 컨텍스트가 있어야 AI 분석/페르소나 비평이 **이 팀의 상황에 정확히 맞게** 작동.

## 8. 카카오 통합 (시장 반응도 20% 직결)

| 지점 | 사용 |
|---|---|
| 로그인 | **카카오 OAuth 단일 로그인** (이메일 가입 없음) |
| 팀원 초대 | **카톡 공유 SDK** — 팀장이 톡으로 초대 링크 보냄 |
| 외부 공유 | 완성된 기획안을 카톡으로 공유 → **"좋아요 누르러 가기"** CTA |
| 알림 | 팀원 피드백 도착 시 카톡 메시지 (비즈니스 채널 비용 검토) |

**왜 중요한가**: 아이디어톤 심사의 **20%가 외부인 포함 좋아요**. 팀이 카톡 한 번에 친구에게 좋아요 부탁할 수 있어야 한다. 우리 서비스 자체가 그 흐름을 도와주면, **우리 사용자 = 우리 팬**이 된다.

**우리의 정체성**: "여기서 끝, 그 다음은 익숙한 도구로."

| 출력 형태 | 사용처 | 출시 시점 |
