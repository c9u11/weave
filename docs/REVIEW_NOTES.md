## A. 사용자가 제기한 이슈 (5건)

### A1. 주제 부합성 — "AI가 [ ]를 없앨 수 있다면?"
**문제**
- 현재 기획은 "AI가 무엇을 없애는가"가 한 줄로 떨어지지 않음.
- "두서없음", "무결론 회의", "0→1 막막함" 등 후보가 섞여 있어 심사위원에게 메시지가 분산됨.

**개선 방향 후보**
- (a) "AI가 **회의의 무결론**을 없앤다" — 피드백 구조화·합의 도출에 초점
- (b) "AI가 **0→1 구간의 막막함**을 없앤다" — 정리·산출물 생성에 초점
- (c) "AI가 **팀 합의 비용**을 없앤다" — 비동기 협업에 초점

→ **하나로 좁히고 슬로건/핵심 기능을 그 슬로건에 맞춰 재정렬해야 함**

---

### A2. AI 활용 깊이 부족 — 단순 프롬프트 엔지니어링 수준
**문제**
- 현재 5개 기능 모두 "텍스트 입력 → LLM 호출 → 텍스트 출력". 심사 기준상 약함.

**깊이를 더할 수 있는 방향**
1. **멀티 에이전트 / 페르소나 시뮬레이션**
   - "엄격한 심사위원", "회의적인 팀원", "낙관적 PM" 페르소나 AI가 각자 비평 → 팀원이 빠진 시점에도 시뮬레이션된 피드백 제공
2. **임베딩 기반 아이디어 클러스터링**
   - 팀원 3명이 비슷한 아이디어를 다른 표현으로 적었을 때 자동 병합·중복 탐지
3. **피드백 컨플릭트 자동 탐지 + 중재안 생성**
   - "A는 실현 가능성 높다고, B는 낮다고 했음" → 양쪽 근거 비교 + 검증 질문 자동 생성
4. **RAG로 유사 사례·우수 기획안 참고**
   - 공개된 우수 공모전 수상작·해커톤 결과물을 벡터DB에 넣어 "비슷한 시도가 있었음/없었음" 자동 답변
5. **멀티모달 입력**
   - 손글씨 메모 사진, 음성 브레인스토밍 녹음 업로드 → STT/Vision으로 자동 정리
6. **반복적(Iterative) 정리**
   - 1차 정리 후 "이 부분이 약합니다" 자가 비평 루프(self-critique) → 자동 보완
7. **평가 기준 자동 학습**
   - 사용자가 직접 입력한 심사 기준에 맞춰 가중치 조정 (예: "실현 가능성 40%면 그 기준으로 분석 강화")

→ **최소 2개 조합 권장** (예: 클러스터링 + 컨플릭트 탐지, 또는 페르소나 시뮬레이션 + 자가 비평 루프)

### A3. 온보딩 과정 부재
**누락된 단계** (팀원 초대 *전*)
- 프로젝트 주제·키워드 (대회명, 도메인)
- 진행 기간 / 마감일 (해커톤 48h vs 학기 과제 2주)
- 무/유료 모델 의향
- **심사 기준 입력** (대회별로 다름 — 사용자 입력 가능해야 함)
- 팀 규모, 역할 구성 (개발자 N명, 디자이너 N명…)
- 산출물 형태 (PPT? 데모? 보고서?)
- 톤·언어 (한국어/영어, 캐주얼/포멀)

**왜 중요한가**
- 이 컨텍스트가 있어야 A2의 AI 분석이 의미 있어짐 (예: 발표 가능성 판단 시 심사 기준 가중치 적용).

---

### A4. 프로세스 상세화 부족
**현재 누락**
- 아이디어 수집 방식: 동시 입력? 순차? 익명? 실명? 1인 1개? 1인 N개?
- 투표 방식: 단일 투표? 가중치? 선호 순위? 동점 처리?
- 투표 기준: A 단계의 4가지(종합/실현/발표/차별) 외에 사용자 정의 가능?
- 피드백 마감: 시간 기반? 전원 완료 기반? 팀장 강제 마감?
- 진행 단계 잠금: 피드백 단계에서 새 아이디어 추가 가능?
- AI 정리 결과 거부/재생성: 몇 번까지? 누가 결정?
- 최종 선정 후 수정: 어디까지 허용?

→ **상태 머신(state machine) 다이어그램 수준으로 정의 필요**

### A5. 카카오 인증·공유 통합
**결정 사항**
- 로그인: 카카오 OAuth 기본 (이메일 옵션은 추후)
- 초대 링크: 카카오톡 공유 SDK 사용 (피드 형태 또는 메시지 템플릿)
- 알림: 카카오 알림톡? 또는 카톡 메시지? (비즈니스 채널 필요 — 비용 검토)

**고려 사항**
- 카카오 비즈니스 등록 / 도메인 검수 / 앱 키 발급 일정
- 미가입자 클릭 시 플로우 (가입 → 초대된 팀 자동 합류)
- 데스크톱 사용자 (카톡 PC만 있을 때 OAuth 흐름)

## B. 추가로 고려가 필요한 사항 (제가 발견한 것들)

### B1. 권한·역할 구조
- 팀장(호스트) vs 팀원 권한 구분이 정의되어 있지 않음
- 팀장만 가능한 액션: 팀 생성, 단계 전환, 마감, 팀원 강퇴, 산출물 확정?
- 팀원 모두 동등한 권한이라면 단계 전환은 다수결?

### B2. 익명 vs 실명 피드백
- **익명**: 솔직한 피드백 가능, 단 무책임한 비판 우려
- **실명**: 책임감 있음, 단 친한 사이에 솔직하기 어려움
- **하이브리드**: 동의·평가는 익명, 자유 코멘트는 실명? 또는 토글?

### B3. 동시성·실시간성
- 여러 팀원이 동시에 입력/투표할 때 충돌 처리
- 실시간 반영(WebSocket) vs 폴링 vs "새로고침" 안내
- 0→1 도구는 비동기가 기본이므로 **실시간 미요구**가 합리적일 수 있음

### B4. AI 결과물 신뢰성
- 할루시네이션·잘못된 정리 발생 시 사용자 대응 흐름
- "이 부분 다시" 부분 재생성 vs 전체 재생성
- AI 출력에 대한 사용자 평가(👍/👎) 수집 → 이후 개선

### B5. 데이터 프라이버시·IP 보호
- 아이디어가 곧 IP. 서버에 저장된 아이디어 유출 우려
- LLM 호출 시 외부(Anthropic/OpenAI)로 전송됨 — 민감 정보 마스킹? 사용자 동의?
- 팀 종료 후 데이터 보관 기간·삭제 정책

### B6. 빈 상태(Empty State)·콜드 스타트
- 첫 사용자가 막막하지 않도록 **샘플 팀/예시 아이디어** 제공
- "예시 보기" 버튼 또는 데모 모드

### B7. 모바일 우선 UX
- 대학생 타깃이므로 모바일 비중 매우 높음
- 카톡에서 링크 → 모바일 브라우저 → 입력까지 **3탭 이내** 목표
- PWA / 반응형 우선

### B8. 시간 압박 시나리오
- 해커톤 48시간처럼 시간 제한이 있을 때 카운트다운·자동 마감 기능
- 마감 임박 시 알림 강도 차등

### B9. 진행률·가시화
- "5명 중 3명 피드백 완료" 같은 진행 인디케이터
- 단계별 체크리스트로 팀이 어디까지 왔는지 한눈에

### B10. 솔로 모드 / 팀원 미응답 대응
- 팀원이 응답하지 않을 때 페르소나 AI가 대신 피드백 (A2-1과 연계)
- 1인 사용 시에도 가치 제공 (다양한 AI 페르소나가 팀원 역할)

### B11. 아이디어가 여러 개일 때
- 1팀 1아이디어 가정인지, 여러 아이디어 중 선택인지
- 페르소나 B(공모전 48시간, 아이디어 3개)는 명확히 다중 아이디어 케이스
- → **다중 아이디어 비교·병합** 흐름이 필요

### B12. AI API 비용 감당
- Free 플랜에서 LLM 호출 비용을 어떻게 감당할지
- 호출 횟수 제한? 모델 다운그레이드(Haiku)? 광고? 학교/기관 스폰서?

### B13. 산출물 버전 관리
- PRD를 N번 재생성했을 때 이전 버전 보존
- 팀원이 직접 편집한 내용과 AI 생성본의 머지·diff

### B14. 다국어
- 영어 발표 준비 시나리오 — 산출물을 영어로 번역 출력
- 외국인 교환학생 팀원 대응

### B15. 거버넌스·심사위원 모드 (옵션)
- 대회 심사위원이 직접 들어와서 평가하는 모드
- "심사위원 페르소나"가 아닌 실제 심사위원

### B16. 실패 사례 / 폐기 처리
- 아이디어를 폐기하거나 팀이 해체될 때 흐름
- 산출물만 개인 보관소로 이전?

### B17. 차별화의 외부 가시성
- 심사위원 입장에서 "이건 ChatGPT로도 되잖아?" 라는 반박을 어떻게 차단할지
- → A2의 깊이 있는 AI 활용 + 팀 협업 구조화가 핵심 답변
- 데모 시 **"ChatGPT 단독으로는 못하는 것"** 한 장면 명확히

### B18. 명칭·브랜딩
- 서비스 이름이 "AI 팀 프로젝트 매니저"로 임시. 슬로건과 함께 정식 네이밍 필요
- 도메인 가용성·상표 검색 사전 체크
